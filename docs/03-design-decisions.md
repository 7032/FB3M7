# 設計判断・確定仕様

ユーザー確認により確定した方針の記録。

## 確定事項（2026-05-30）

| 項目 | 決定 | 備考 |
|---|---|---|
| 拡張名 | **FB3M7**（独自の造語名） | 商標「F-BASIC」を製品名に直接使わない。`FB3`=F-BASIC 3 の略号 + `M7` を組み合わせた造語。説明文では対応言語として記述的に言及するに留める |
| 対象エディタ | VS Code 拡張 | Qiita 記事ベースの構成 |
| 実装言語 | **TypeScript** | 型安全・保守性 |
| ファイル拡張子 | `.BAS` | |
| 文字コード | **UTF-8 基本** | Shift-JIS は将来課題 |
| 対象方言 | **FM-7 F-BASIC V3.0 標準** | 77AV 拡張は将来 |
| 自動採番の操作 | **Enter で次番号を自動挿入** | 増分 10。一括採番コマンドも別途用意 |
| ドキュメント | `docs/` 以下に Markdown 蓄積 | |
| バージョン管理 | GitHub `7032/FB3M7`（public） | 作業前に Issue 作成 / push は明示指示時のみ |

## 実装上の判断

- **自動採番は keybinding 方式**: `onEnter` コマンドを `enter` にバインドし、`editorLangId == fbasic` かつ補完ウィジェット非表示等の `when` 条件で限定。BASIC 以外・補完中の Enter を阻害しない。
- **Renumber はドキュメント全体を一括置換**: 行ごとにコメント部を温存しつつコード部のみ書き換え。文字列内・コメント内の数値は対象外（`findCommentStart` が文字列状態を追跡）。
- **ジャンプは DocumentLink + Definition の二本立て**: クリック（DocumentLink）と F12/Ctrl+Click（Definition）の両方に対応。リンク化は「参照キーワード後続の数値」かつ「実在する行番号」に限定し誤検出を抑制。
